Preventing Cross-Site Scripting (XSS)

Cross-Site Scripting (XSS) is a vulnerability that allows attackers to inject malicious scripts into a webpage. These scripts can execute in the browser of unsuspecting users, compromising their data and security.

In our project, we identified a potential XSS vulnerability in the "Filtered Results" section, where turtle data is dynamically added to the page. If the `innerHTML` property is used to insert content, it could execute harmful scripts, such as:

<script>alert('Hacked!')</script>

To address this, we use the `textContent` property for untrusted data (like turtle names and descriptions). This ensures the data is treated as plain text, preventing scripts from being executed.
